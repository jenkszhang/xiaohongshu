# 小红书推流机制（深度笔记）

> 沉淀日期：2026-05-17
> 用途：这不是"运营公式"，是工程内所有 [[xhs-*]] skill 的**底层认知**。所有 SOP（5 处布点、封面 hook 公式、首评 30 分钟、共性矩阵）都是这套机制的具体动作翻译。理解了下面这些，你能自己判断每一条 SOP 是不是还活着；不理解，就只能盲跟。
>
> **一句话**：算法不奖励"好"，奖励"用户花时间且愿意互动"。两者常重合但不等同 —— 这是"很有价值但没流量"和"很水但能爆"的错位本质。

---

## 0. 两套语言

聊小红书算法，外界有两套语言体系，先分清楚：

| 视角 | 来源 | 可信度 | 用法 |
|---|---|---|---|
| **工程视角**：四段漏斗 + 多目标融分 | 小红书搜推算法负责人王树森 B 站公开课 / CSDN 技术复盘 | 🟢 高（平台内部人讲）| 作底层认知 |
| **运营视角**：CES 评分 + 流量池跃迁 | MCN / 运营圈反推的经验模型 | 🟡 中（平台从未官方确认）| 作精排融分的简化代理使用 |

CES 公式 `点赞×1 + 收藏×1 + 评论×4 + 转发×4 + 关注×8` **不是平台原话**，但方向正确——深度互动权重高于轻互动。可以当工程侧多目标融分的简化代理用，但别迷信具体系数。

## 1. 工程视角：四段漏斗

```
[亿级笔记池]
     │
     ▼ 召回（几十条独立通道并行取回，每条几十~几百篇）
     │  ItemCF / Swing / UserCF / 双塔向量 / 关注通道 / 热门池 / 冷启动池
     │  ⚠ 这一关没大模型，靠的是用户向量 × 笔记向量的相似度
     │
[千级候选]
     │
     ▼ 粗排（三塔模型，第一次预估互动率）
     │  几千 → 几百
     │
[百级候选]
     │
     ▼ 精排（决定排序的核心阶段）
     │  大规模深度神经网络 MMoE 类
     │  同时预估 7+ 个目标：pCTR / 点赞率 / 收藏率 / 完播率 / 评论率 / 转发率 / 关注率 / 负反馈率
     │  融分公式合成最终分数
     │  ⚠ 精排不做截断
     │
     ▼ 重排
     │  · 多样性抽样（MMR / DPP）
     │  · 规则打散（同作者/同话题不挨着）
     │  · 插入广告 + 运营内容
     │  ⚠ 精排打分高不等于首屏 —— 重排会强行打散
     │
[最终展示]
```

**对工程的最重要含义**：
- **召回阶段**：tag 和正文关键词决定笔记向量 —— 这是 [[xhs-copy]] §5 「5 处布点」红线的底层依据
- **精排阶段**：单项指标突出但其他项拉胯 = 融分低 —— 这是 [[xhs-data-analyze]] §5 「共性矩阵 7+ 维度全检测」的底层依据
- **重排阶段**：算法主动打散同主题 —— 所以"一个主题连发 5 篇会自相残杀"，需要给系列内容间留时间和差异

## 2. 精排在预估什么（多目标融分）

对每一篇候选笔记，精排同时预估**当前这个用户**会不会做下面这些动作：

| 目标 | 决定阶段 | 工程内对应抓手 |
|---|---|---|
| **pCTR**（点击率）| 封面 + 标题 | [[xhs-image]] §5.1.0 封面 hook 公式（数字 + 主长尾词 + 反差结论）|
| **完播率 / 阅读完成率** | 进入笔记后 | [[xhs-copy]] §6 纯文本流水读完 + 正文有节奏分段 |
| **点赞率 / 收藏率** | 阅读过程中 | 内容类型决定上限：教程/清单收藏天然高、情绪类点赞天然高 |
| **评论率** | 阅读结尾 | [[xhs-copy]] §X 评论钩子 + [[xhs-publish]] §3 首评起话头 |
| **转发率** | 阅读结尾 | 投票贴纸（[[xhs-copy]] §X）/ 可代入场景 |
| **关注率** | 看完后 | 人设连贯性 → 跨多篇累积（账号视觉锁定 + STYLE_PREAMBLE）|
| **负反馈率** | 任何时刻 | 封面承诺要在首句兑现，不要骗点击 |

**冲突是真实的**：
- 标题党 → 拉高 pCTR，但拉低完播率和点赞率 → 融分反而低
- 纯干货堆砌 → 收藏率高，但评论率低 → 融分也低
- 强情绪输出 → 评论多，但收藏少 → 看类型上限

**结论**：算法要的是「这个内容类型该有的指标都达到中位数以上」，不是「单项 P99 + 其他 P10」。

## 3. 按用户浏览动作拆解可拉高的内容特征

精排七目标都是顺着用户浏览动作展开的。所以**怎么写**就是顺着这个动作链路逐段对齐。

### 3.1 封面 + 标题（决定 pCTR）

- 小红书是**双列瀑布流**，CTR 是用户独立行使的过滤器 —— 算法把你推到首屏，但用户不点，后续点赞/收藏/停留全是 0，笔记就死了
- CTR 5% vs 10% = 后续推流 2× 差距
- 高 CTR 封面特征：信息密度够 / 视觉锚点（人脸·对比·数字·箭头）/ 文字大且可读 / 对得上搜索词或兴趣词
- 标题前 18 字最关键 —— 双列展示是截断的，后面的字看不到

**↳ 工程对应**：[[xhs-image]] §5.1.0 封面 hook 公式 = `数字 + 主长尾词 + 反差结论`。
老号那篇 8.7w 浏览的「9 点的咖啡，等于白喝」就是这个公式（数字 9 点 + 长尾词咖啡时机 + 反差结论"白喝"）。

### 3.2 前 3 秒（决定停留 + 负反馈）

- 用户点进来后前几秒决定是返回还是继续看
- **返回得快 = 强负信号**，精排会把这篇压回去
- 封面承诺什么，开头就给什么，**不能把干货藏在最后让用户翻**

**↳ 工程对应**：[[xhs-copy]] §5 五处布点第 2 处「正文首句必带主长尾词」 = 第一段就兑现封面承诺 + 主题词复现。

### 3.3 阅读过程中（决定完播率 + 收藏率）

- **完播率/阅读完成率** 是近 1-2 年权重明显上升的目标 —— 比点赞难造假
- 高完播特征：
  - 结构清晰、有节奏（问题→方法→例子→总结，或编号列表，让用户能预期还剩多少）
  - 视频开头不要废话，前 3 秒说清楚在讲什么
  - **长度匹配价值** —— 3 分钟能讲完的事强拍 8 分钟，完播必崩
- 高收藏特征：有"留下来以后用"的价值 —— 教程 / 清单 / 攻略 / 模板 / 测评结论。纯情绪输出点赞高但收藏低

**↳ 工程对应**：[[xhs-copy]] §6 纯文本约束 + 末行 inline tag = 让正文像流水一样一气读完；图文清单类要做"留得下"的实操干货。

### 3.4 互动阶段（决定评论率 + 转发率 + 关注率）

- **能拉评论的钩子**：
  - 开放式结尾：「你们觉得呢」「有没有人和我一样」「求推荐第二家」
  - **故意留白或不完美** —— 完美攻略反而没评论，带点缺陷或主观判断让人想补充
  - 反共识观点 —— 但要小心翻车
  - **可代入的具体场景** —— 让读者觉得"这就是我"，自然想留言分享自己的版本
- **关注率**是最难拉的目标 —— 它要求用户对**作者**而不是这一篇产生兴趣
  - 人设连贯的账号 > 每篇都爆但风格漂移的账号
  - 后者每篇都是新冷启动，前者每篇都在累积关注转化

**↳ 工程对应**：
- [[xhs-publish]] §3.1 首评必带主长尾词 + 钩子型话头 = 同时拉评论率 + 5 处布点第 5 处
- 双号视觉锁定（A-米白暖橙 vs F-雾蓝鼠尾草绿）+ STYLE_PREAMBLE 强制 = 人设连贯的视觉抓手

### 3.5 搜索匹配（决定长尾流量）

**这是小红书跟抖音最大的差异**：

- 业内推测 60-70% 流量来自搜索（平台未公开过具体数字，作量级参考）
- 搜索流量判定逻辑跟推荐**完全不同**：关键词匹配度 × 笔记历史互动
- 含义：一篇笔记推荐流量两周后归零，但**在搜索侧仍持续出现** —— 这是小红书内容长尾价值远高于抖音的根本原因
- 商业意图最高的流量在这里 —— 用户主动搜索 = 购买决策前
- 实操：标题塞 1-2 核心搜索词、正文前 200 字自然重复、tag 选中等热度（头部词排不上）

**↳ 工程对应**：[[xhs-copy]] §5 主长尾词 5 处布点红线（标题前 10 字 / 正文首句 / 中段 2-3 次 / tag ≥ 2 / 首评）—— **全面对齐召回向量化 + 搜索匹配**。
这是工程内最贵的一条红线，因为它一次性同时抬高了召回成功率 + 搜索流量。

## 4. 五个反直觉点

### 4.1 双列瀑布流让封面权重远超抖音

- 抖音单列沉浸式 = 算法控制权大
- 小红书双列 = CTR 是用户独立过滤器
- 博主对封面执念不是审美问题，是**机制问题**

### 4.2 限流大多不是"惩罚"，是判定

- 真正的惩罚性限流（违规 / 导流 / 敏感词）很少见
- 多数"被限流" = **精排分数低 + 召回没匹配** 的自然结果
- 排查方向（不是诉苦平台不公）：
  - tag 不匹配 → 召回失败
  - 封面 CTR 太低 → 精排分数压低
  - 初始互动率没达到该兴趣簇均值 → 同池竞品挤掉

**↳ 工程对应**：[[xhs-data-analyze]] §5.4 翻车反向规律 = 找系统性原因（陈述标题 + 早发 + 无破绽），而不是诉诸"被限流"。

### 4.3 互动质量 > 互动数量

- 近 1-2 年观察：高价值评论、完播率成为隐性指标
- 纯收藏无评论可能被判低质
- 平台主动加入"难造假的目标"（长评论、收藏后回访、关注转化）来对冲诱导点赞
- **求互赞群刷数据效果越来越差** —— 只能拉单一指标，融分动不了

### 4.4 好内容的定义随兴趣簇而变

- 母婴用户重信息可靠性 / 潮玩用户重视觉冲击 —— **算法只对兴趣簇内的相对表现打分**
- 观察对标账号、对标笔记 >> 研究"通用爆款公式"

**↳ 工程对应**：[[xhs-data-analyze]] §2 爆款定义用"账号自身分布 P80"而非绝对值 = 同一逻辑。

### 4.5 精排打分高 ≠ 上首屏

- 重排会做多样性抽样 + 规则打散
- 同作者 / 同话题不能挨着展示
- 所以**一个主题连发 5 篇会自相残杀**，需要主题间错峰

## 5. 五个容易被忽略的执行点

### 5.1 初始互动率比绝对数量重要

- 算法看的是 500 曝光里多少人互动，不是 100 vs 1000 赞
- **5000 粉精准账号可能比 50 万粉泛账号更容易爆**
- 因为后者初始曝光给的是泛粉丝，互动率反而上不去

### 5.2 发布后 1-2 小时数据决定上限

- 精排会持续用新数据更新预估，但**前 1-2h 权重最高**
- 算法要尽快判断这篇内容的"潜力"，时间拖太久就过气了
- 凌晨 3 点发 = 错过算法判定窗口

**↳ 工程对应**：
- [[xhs-publish]] §3 首评 30 分钟黄金窗口 = 前 1-2h 数据窗口的具体抓手
- [[xhs-data-analyze]] §5.2 共性矩阵的"发布时段"维度 = 找到本账号的算法判定窗口

### 5.3 账号垂直度影响召回匹配

- 系统按历史内容给账号打标签
- 换赛道 = **召回失配**（老粉标签匹配不上 + 新受众标签也匹配不上）
- 垂直 ≠ 重复 ≠ 单一话题 —— 是**同一受众群体的不同切面**

**↳ 工程对应**：双号策略 = 老号继续泛健康散点（不动）+ 新号「上班族-抗炎饮食」聚焦垂直 = 让新号被算法快速识别为"做抗炎的"，召回向量稳定。

### 5.4 别迷信单一指标

- "点赞重要还是收藏重要" 这个问题本身是错的
- 算法看融合分，要的是该类型内容**中位数以上的多指标平衡**

### 5.5 内容类型决定指标天然上限

- 教程类收藏天然高、情绪类点赞天然高、争议类评论天然高
- **不要跨类型对标** —— 拿教程类去比情绪类的点赞数，是错的对标

**↳ 工程对应**：[[xhs-data-analyze]] §5 共性矩阵分维度比较"爆款集合 vs 全账号分布"，而不是跟外站绝对值比。

## 7. 实操原则（一句话提炼）

> **别盯着 CES 公式凑数据，盯着精排的多目标做内容**：封面让人想点、开头让人看完、结尾让人想评论、整体让人想关注。机制设计就是冲着这个去的。
