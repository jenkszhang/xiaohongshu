## 两文档关系

| | 推流机制 | 利他思维 |
|---|---|---|
| 解决问题 | 算法把笔记推给**对的人** | 对的人看到后**愿意点 / 完播 / 收藏 / 评论 / 关注** |
| 抓手 | 5 处布点 / 封面 hook 公式 / 首评 30 分钟窗口 | 3 问自检 / 封面图清单化 / 副图动作链 / post.md 必填段 |
| 类型 | 机制理解（黑箱推测）| 叙事原则（直接可操作）|
| 是否红线 | 通过子规则间接生效 | **红线 #9 显式生效** |

两者叠加：推流机制是"算法层 + 流量入口"，利他思维是"叙事层 + 留存转化"。**任何一篇笔记都要同时通过两套自检**才算合格。

## 维护原则

1. **不写「怎么做」**——那是 SKILL.md 的工作
2. **写「为什么这么做」**——把算法/平台/用户行为机制讲清楚
3. **每条机制都要给出 ↳ 工程对应抓手**——否则就是空理论
4. **每半年校验**——平台算法在变，王树森也在更新公开课
5. **数据 > 文档**——工程数据跟文档预测不符时，信数据、改文档
