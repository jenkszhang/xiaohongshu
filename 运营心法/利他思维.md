# 利他思维（工程内最终自检红线）

> 一切内容元素必须从「**读者能拿走什么**」出发，不是「我想表达什么 / 我多懂」。
> 这是叠加在 [[xhs-copy]] 5 处布点 + [[xhs-image]] hook 公式之上的**最终自检**——前者保证算法把笔记推给对的人，利他思维保证对的人看到后**愿意点 / 看完 / 收藏 / 评论 / 关注**。
> 对应 [[运营心法/推流机制]] §2 多目标融分——利他直接抬升 pCTR + 完播 + 收藏 + 评论 + 关注 5 个目标的天然上限。

---

## 1. 一句话定义 + 3 问自检公式

每个内容元素（标题 / 封面 hook / 封面图 / 每张副图 / 首句 / 中段 / 结尾 / 首评）出来后，逐个问：

1. 这句话/这张图的**主语**是「我」还是「你/姐妹/打工人」？（利他要"你"）
2. 读者**立刻能拿走的具体动作**是什么？（一句话讲不出来 = 这块没利他）
3. 读者看完是觉得「**博主好厉害**」还是「**我能用上**」？（利他要后者）

任何一个偏第 1 种答案 → 重写。

---

## 2. 利他 ≠ 不讲我

利他不是「不能讲我自己」，而是「**讲我的目的是为了帮你**」。

| ✅ 利他（讲我 = 减少你的工作量）| ❌ 利己（讲我 = 炫资历）|
|---|---|
| "我替你筛过了 50 个外卖品类，你照着点就行" | "我研究了半年终于搞懂抗炎" |
| "试错 3 次踩雷的我，整理了这份避坑表" | "作为专业营养师告诉你..." |
| "我之前也不会看标签，后来发现记 1 张表就够" | "学营养学 5 年终于明白" |

## 3. 五个层面的利他 vs 利己（锚：「抗炎清单」）

### 3.1 标题

| | 例子 | 评 |
|---|---|---|
| ❌ | 「我的外卖抗炎食谱分享」 | "我的""分享" 博主中心 |
| ❌ | 「营养师亲测半年抗炎清单」 | 炫资历 |
| ✅ | 「外卖照点这 8 样，少炎症 80%」 | 主语是动作（你照点）+ 量化收益 |
| ✅ | 「不会看营养标签也能抗炎，外卖照这清单点」 | 显式承认读者痛点 |
| ✅ | 「打工人外卖抗炎清单，存手机点餐时翻」 | 直接告诉用户怎么用 |

主长尾词、5 处布点、hook 公式**全部不变**，只换主语和叙事视角。

### 3.2 封面 hook（公式不变，主语变）

hook 公式仍是 [[xhs-image]] §5.1.0：`数字 + 主长尾词 + 反差结论`。**内容主语**必须是「你的动作 + 你的收益」：

| | 例子 |
|---|---|
| ❌ | 「我研究 1 年的抗炎清单」 |
| ✅ | 「点外卖前 30 秒，少吃 80% 促炎」 |
| ✅ | 「8 样外卖，照点就抗炎」 |

### 3.3 封面图本身（工程之前完全没考虑）

**封面要变成「读者第一张可保存的清单」** —— 直接把对照表 / 替代公式 / 黑白名单的核心 4-6 项做成封面图，让用户**看到封面那一刻就有"截图就行"的冲动**。

跟雾蓝北欧风（[[STYLE_PREAMBLE]]）/ 暖橙手绘水彩**不冲突**——杂志感和治愈感本来就讲究信息密度 + 排版清晰。区别只是：
- ❌ 旧做法：封面是博主露脸 / 一碗精致食物 + 一句 hook 文字
- ✅ 新做法：封面是浓缩的清单表 / 对比图 + hook 文字

### 3.4 副图序列：每张回答一个「我下一步该怎么做」

工程旧组合是「场景 / 公式 / 金句 / 故事」，新组合是 **「读者动作组合」**：

| 副图位 | 旧（散点）| 新（利他动作链）| 读者拿走 |
|---|---|---|---|
| 02 | 场景图 | 黑名单 | 避什么 |
| 03 | 公式图 | 绿灯名单 | 选什么 |
| 04 | 解剖图 | 替代公式（X→Y）| 一对一替换 |
| 05 | 金句图 | App / 场景里怎么操作 | 进 App 怎么做 |
| 06 | 故事图 | 1 周示范 / 截图提醒 | 一周不用想 |

**每张副图必须能用一句话答出「读者拿走 → ___」**，答不出来的那张就是装饰，重做。

### 3.5 正文

| 段落 | 利他做法 |
|---|---|
| **首句** | 承认读者痛点 + 给收益（不要"作为... 半年，今天分享"）|
| **中段** | 直接上干货清单，不要讲机理（机理 = 利己 / 清单 = 利他）|
| **倒数第二段** | 显式利他声明：「我替你筛过了 X」「试错 3 次替你避了 Y」—— 把"我做了功课"翻译成"你不用做功课" |
| **结尾** | ① 合理化收藏（"截图存手机，下次 X 时看一眼"）② 利他承诺型评论钩子（"评论区告诉我 X，我帮你 Y"）|

### 3.6 首评（5 处布点第 5 处也是利他点）

首评模板 = **共情（我之前也...）+ 利他承诺（我帮你 / 我整理了第二份）** + 主长尾词。

## 4. 四种「伪利他」陷阱

| 陷阱 | 表面 | 实质 |
|---|---|---|
| **鸡汤利他** | 「你值得最好的」「爱自己从抗炎开始」 | 没有具体动作 |
| **资历利他** | 「专业营养师告诉你..." | 主语还是我 |
| **强行钩子** | 「评论区告诉我你的想法」 | 没有承诺，纯索取 |
| **干货堆砌** | 列 100 条规则 | 读者无法选择 = 等于没给 |

## 5. 跟现有红线的关系（叠加，不替代）

| 红线 | 解决的问题 | 算法阶段 |
|---|---|---|
| **5 处布点**（[[xhs-copy]] §5）| 召回 + 搜索能否找到你 | 召回 + 搜索 |
| **封面 hook 公式**（[[xhs-image]] §5.1.0）| CTR 够不够 | 用户决策第 1 秒 |
| **利他自检**（本文档）| 点进来之后愿不愿意完播/收藏/评论/关注 | 用户决策之后所有秒 |

三条红线**叠加生效**。利他自检是**最后一关**：5 处布点过了 + hook 过了 + 利他过了 = 全套 SOP 通过。
